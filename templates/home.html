How It Works

KnownPlainText, which was built and is maintained by AverageSecurityGuy, uses a database to store LM and NTLM password hashes and their associated plaintext equivalent, all of which meet the Windows complexity requirements and are at least six characters long. Users upload the LM and NTLM hashes to the KnowPlainText server and the server returns the plaintext password for any hashes stored in the database. Hashes that are not in the database are cracked offline and put into the database for future use. In addition, the database is regularly purged of unrequested hashes so that overtime, the database only contains the most commonly requested complex passwords.

Why It Works

It is common for a person to reuse passwords across services and it is also common for multiple users in a large group to use the same password. Over and over again, Internet password breaches have shown us that the same 100 passwords, or variations of them, account for a significant percentage of user passwords. Since password reuse is so common, it is not necessary to store terabytes of hash/plaintext value pairs because a small subset will yield significant results.

My Password Was Not Found

KnownPlainText will not find every password, because it is physically impossible to store every password. Fortunately, human nature shows that we only have to store a relatively small number of passwords to yeild effective results. KnownPlainText is expected to have a low crack rate in the beginning, but over time, as more users submit hashes and they are cracked and analyzed for patterns, the database will become more effective.

Why Should I Use the Service?

You can use rainbow tables to crack LM hashes and recover nearly 100% of the plaintexts but it will take you hours. You can also use hashcat or JtR and get very high results but you waste a lot of CPU cycles testing passwords that do not meet the Windows complexity requirements and are therefore doomed to fail. KnownPlainText strives for a 10-20% crack rate in seconds not minutes or hours. This allows you to spend less time cracking passwords and more time popping boxes. The average pentester charging between $100 to $200 per hour would only need to save 5-10 hours a year to pay for the service.
